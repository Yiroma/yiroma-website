# Conventions du projet Yiroma

Ce dossier regroupe l'ensemble des conventions adoptées pour le projet. Chaque fichier couvre un domaine spécifique.

## Sommaire

1. [Git](./01-git.md) — Branches, commits, versioning
2. [Code Style](./02-code-style.md) — Formatage, nommage, outils (Prettier, ESLint, TypeScript)
3. [Frontend](./03-frontend.md) — Next.js, React, Tailwind, structure des dossiers, composants
4. [Qualité](./04-qualite.md) — Accessibilité, SEO, performance, sécurité, principes (DRY, KISS, YAGNI)

## Règle générale

Toute intervention sur le projet doit respecter ces conventions.
En cas de doute, se référer au fichier correspondant ou au [design system](../design/design-system.md).
